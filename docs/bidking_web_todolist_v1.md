## 2. 当前已经明确的玩法规则

### 2.1 局外结构
- 登录后主界面包含：`背包 / 仓库 / 商店 / 竞拍`
- `背包` 当前先不作为核心玩法
- `仓库` 用于存放藏品，并支持直接出售
- `商店` 用于购买消耗型道具

### 2.2 竞拍入口
- 选择 `竞拍` 后进入地图选择
- 地图需要校验玩家是否有足够资金进入
- 进入地图后可选择：
  - `角色`
  - `5 种不同类型道具`
- 道具为全角色通用
- 单局每种道具最多带 `1 个`
- 局外可拥有多个同类道具库存

### 2.3 对局结构
- 每局共 `4 名竞拍者`
- 当前阶段除玩家外其余均可由 `AI` 代替
- 进入对局时，随机选择当前地图下的一个 `子场景`
- 子场景影响物品类别倾向，不直接播报信息

### 2.4 页面结构
- 左中区域：信息汇总区
- 右侧：4 个竞拍者信息区
- 主区域：盲盒展示区
- 底部：出价与道具按钮区

### 2.5 信息规则
- 每回合固定公布 `1 条公共信息`
- 角色信息按回合 `自动触发`
- 角色信息：`仅自己可见`
- 道具使用行为：`全员可见`
- 道具揭示结果：`仅自己可见`

### 2.6 盲盒显示规则
- 开局盲盒区域 `什么都不显示`
- 若只知道品质：显示 `1 格颜色块`
- 若只知道轮廓：显示 `白色轮廓`
- 若轮廓与品质都知道：显示 `真实轮廓 + 对应颜色`

### 2.7 回合行为
- 每局最多 `5 回合`
- 每回合玩家最多：
  - 使用 `1 个道具`
  - 出价 `1 次`
- 出价提交后，本回合不可修改
- 出价为 `自由输入`
- 出价计算器保留快捷倍乘按钮：`1.2 / 1.4 / 1.6`

### 2.8 拍下规则
- 第 1 回合：`第一名 >= 第二名 × 2.0`
- 第 2 回合：`第一名 >= 第二名 × 1.6`
- 第 3 回合：`第一名 >= 第二名 × 1.4`
- 第 4 回合：`第一名 >= 第二名 × 1.2`
- 第 5 回合：`价高者拍下`

### 2.9 结算规则
- 拍下后进入结算页
- 先展示所有藏品轮廓
- 再按品质延迟揭示完整信息：
  - 白色：`0.5s`
  - 绿色：`1s`
  - 蓝色：`1.5s`
  - 紫色：`2s`
  - 金色：`3s`
  - 红色：`5s`
- 结算页实时统计盈亏
- 藏品实际上已进入仓库
- 结算页提供“按颜色立即卖出”快捷操作
- 出售勾选规则：勾选什么颜色，就卖出什么颜色

## 3. 按当前文档能否直接做 Web 玩法版

结论：`可以做一个完整可玩的 Web 前端原型版，但还不能做成长期运营版。`

理由：
- 已有地图文档、角色文档、物品文档，核心玩法主循环已经存在
- 当前可以实现：
  - 地图选择
  - 角色选择
  - 道具带入
  - 盲盒信息揭示
  - 五回合竞价
  - AI 出价
  - 结算与仓库入库
- 目前不需要后台也能做纯前端版
  - 本地内存状态即可跑通整局流程
  - 如有需要可先用 `localStorage` 存钱、仓库、商店库存

## 4. 当前文档还不齐全的地方

以下内容不影响做第一版 Web 原型，但会影响实现细节与后续扩展：

### 4.1 道具系统文档缺失
- 缺少完整道具清单
- 缺少每种道具的触发对象、揭示规则、稀有度、商店价格
- 缺少 AI 如何选择道具的规则

### 4.2 竞拍资金规则未完全定稿
- 玩家总资金是否同时承担：
  - 地图准入
  - 竞拍出价
  - 商店买道具
- 竞拍失败后是否有损耗尚未定稿

### 4.3 AI 策略文档缺失
- 只知道“尽量智能一点”
- 缺少：
  - AI 估值逻辑
  - AI 是否保守/激进
  - AI 是否根据角色偏好使用道具
  - AI 如何在不同回合调整出价

### 4.4 页面交互细节未完全定稿
- 同一回合中，公共信息、角色信息、道具使用、出价、结算判定的前端表现顺序需要明确
- 玩家未出价时的状态表现未定
- 多人信息区的具体展示字段未定

### 4.5 仓库与商店规则未完整定义
- 仓库中的藏品后续是否有收藏、图鉴、任务用途未定
- 商店货币、商品刷新、价格机制未定

## 5. 对 Web 实现的可行性判断

### 5.1 纯前端实现是可行的
第一版只做 Web 玩法，可以完全不接后台，采用：
- `HTML + CSS + JS` 或前端框架
- 本地状态管理
- 本地随机种子生成对局
- 本地模拟 AI

### 5.2 第一版不需要 MySQL
如果当前目标只是“把玩法跑起来”，暂时不需要：
- 登录系统
- 后端服务
- MySQL
- 实时多人网络同步

### 5.3 未来可以平滑接后台
等玩法跑通后，可逐步增加：
- 用户系统
- 仓库持久化
- 商店库存持久化
- 战绩记录
- MySQL 数据库存档
